## Project Overview

**PERCEPTION** investigates whether the visual appearance of urban streets (via Street View imagery) and the structural properties of street networks can predict crime risk at the neighborhood level in Lima, Peru.

### Core Innovation

Rather than relying solely on reported crime statistics, the project leverages:

- **Visual embeddings**: 768-dimensional representations from a SimCLR+ViT model trained on Street View photos
- **Structural features**: Graph-theoretic properties (degree, betweenness, closeness, clustering) of the street network
- **Spatial validation**: GroupKFold cross-validation with geographic blocks to eliminate leakage from spatially overlapping neighborhoods

The study area comprises two socioeconomically distinct Lima districts:
- **Barranco**: 400 street intersections, 42.2% labeled as high-crime
- **La Victoria**: 1,450 street intersections, 11.2% labeled as high-crime (highly imbalanced)

---

## Research Question and Hypothesis

**Primary Question**: Can visual and structural properties of urban environments predict crime risk?

**Hypothesis**: The appearance of streets (crowding, commercial activity, infrastructure conditions) and street network centrality (how accessible a location is) are jointly predictive of crime, even after controlling for spatial correlation biases in machine learning models.

**Secondary Question**: How do the relative contributions of visual vs. structural signals differ across urban morphologies?

**Finding**: In compact, homogeneous districts (Barranco), street network structure alone suffices; in heterogeneous, commercially mixed districts (La Victoria), visual signals add significant value.

## Data Sources

### Crime Records
- **Source**: Official crime statistics by Lima district, 2016
- **Format**: CSV with crime incidents, coordinates (lat/lon), category, and annual counts
- **Coverage**: All crime types (assault, theft, etc.) aggregated by location
- **Labels**: Generated via Gaussian Mixture Model (GMM) with dual labeling at node and neighborhood level

### Street Network
- **Source**: OpenStreetMap (OSM)
- **Processing**: Downloaded via Nominatim, snapped to nearest street node (tolerance: 150m)
- **Graph**: Undirected, unweighted (edges represent street segments)

### Street View Imagery
- **Source**: Google Street View Static API
- **Coverage**: 
  - Barranco: 985 crime points with images (out of 1,106 total points)
  - La Victoria: 864 crime points with images (out of 3,807 total points, 22.7% coverage)
- **Per-point sampling**: 12 images (headings: 0°, 30°, 60°, ..., 330°) at 224×224 resolution
- **Image filtering**: Blacklist of corrupted/truncated files generated by `filter_invalid_images.py`

## Methodology

### Step 1: Crime-to-Graph Snapping (`graph_pipeline.py`)

1. Download street network graph from OSM for each district
2. Snap each crime record to the nearest street node (within 150m)
3. Compute crime counts per node and per neighborhood (ego-graph, depth≤3)
4. **Dual GMM labeling**: Fit independent 2-component GMMs at node and neighborhood levels, excluding zero values to prevent collapse of the "safe" component onto zero
5. Output node labels (primary: neighborhood-level, secondary: node-level)

**Decision**: Dual GMM handles high sparsity; many nodes/neighborhoods have zero crime. Fitting while excluding zeros prevents mode collapse while preserving the signal of "very safe" locations.

### Step 2: Street View Embedding Extraction (`extract_embeddings.py`)

1. Load pre-trained SimCLR+ViT checkpoint (trained on 27k images, 50 epochs, best epoch saved)
2. Traverse image directory, parse `crime_id` from folder structure (`{crime_id}.0/heading_*.jpg`)
3. **ID Normalization**: Canonicalize all IDs to format `"19833096.0"` (float suffix) to match disk structure, preventing silent join failures
4. Load and apply blacklist of invalid images (generated by filter step)
5. Extract 768-dim embeddings (ViT backbone CLS token, L2-normalized) for all valid images
6. Save: `embeddings.npy` (N, 768) + `metadata.json` (paths, crime_ids, headings, districts)

**Decision**: Use backbone output (768-dim), not the projector head (512-dim). The projector is downstream and discarded at inference in eval.py anyway.

### Step 3: Neighborhood Tensor Assembly (`Paso4_assemble_neighbourhood_tensors.py`)

For each street node (vecindario):

1. Build adaptive ego-graph: expand depth (1→3) until finding ≥5 distinct crime points **with available images**
2. **Critical fix**: Filter candidate points by availability **before** sampling, not after. This prevents zero-vector dilution: two nodes with shared crime points risk having one filled with zeros if coverage is partial
3. Spatial stratification: use KMeans on lat/lon to select 5 geographically diverse points (not random)
4. Per-point sampling: select 4 images from available pool (with replacement if <4 exist, flag `oversampled_images`)
5. Concatenate into (20, 768) tensor per neighborhood

**Outputs**: 
- `neighbourhood_tensors.npy`: shape (N_valid, 20, 768)
- `coverage_report.csv`: diagnostics (depth used, oversample flags, insufficient coverage flags)
- `node_ids_order.json`: ordering of nodes in tensor (for alignment with labels)

**Decision**: Mismatch between CSV crime IDs and image crime_ids surfaced here as 21.6% overlap in La Victoria. Caused by partial Street View coverage, not a bug—handled gracefully via adaptive depth.

### Step 4: Feature Engineering (`Paso5_eigen_features.py`)

**Baseline comparison**: Test 5 feature aggregation strategies on mean-std normalized embeddings (768→1536 dims: mean + std deviation across 20 images per neighborhood).

1. **Laplacian (raw)**: Compute cosine-similarity affinity on 768-dim, take normalized Laplacian, extract bottom-k eigenvectors (k=5), flatten (5×768=3840-dim). **Result**: Collapsed, AUC≈0.5 in both districts
2. **Laplacian + PCA**: PCA(768→50), then Laplacian on reduced space, flatten (5×50=250-dim). **Result**: Still collapsed
3. **Mean-pooling (raw)**: Simple mean of 20 embeddings (768-dim). **Result**: AUC 0.79 (Barranco), 0.86 (La Victoria)
4. **Mean-pooling + PCA**: Mean of PCA-reduced embeddings (50-dim). **Result**: AUC 0.76/0.84
5. **Mean+Std pooling**: Concatenate mean and std across 20 embeddings (1536-dim). **Result**: AUC 0.79/0.87 (WINNER)

**Why Laplacian failed**: By design, we skip the first (trivial) eigenvector ≈0, losing the "average appearance" signal. This is correct theory (first eigenvector carries no information in spectral methods), but wrong for this problem—the average appearance of a neighborhood IS predictive.

**Cross-validation**: Repeated StratifiedKFold (5 folds × 5 repeats) on spatial data to estimate stability; PCA fit globally on all 30,866 embeddings to avoid overfitting with N_neighborhood=400.

**Output**: `features_mean_std.npy` selected for all downstream tasks.

### Step 5: Multimodal Fusion and Threshold Optimization (`Paso6_final_classifier.py`)

**Three feature sets**:
- Visual only: Mean+Std pooling (1536-dim)
- Structural only: Topological features—degree, betweenness, closeness, clustering, avg_neighbor_degree (5-dim)
- Fused: Concatenation (1541-dim)

**Three classifiers** (per feature set):
- Logistic Regression (C=0.1, balanced class weights)
- Random Forest (300 trees, max_depth=8, min_samples_leaf=3)
- MLP (64 neurons, early stopping)

**Two CV schemes** (per combination = 18 evaluations total):
- **Spatial (PRIMARY)**: GroupKFold with 5 geographic blocks (KMeans on lat/lon). Avoids leakage: neighbors stay in same fold
- **Random (REFERENCE)**: StratifiedKFold with random splits. Inflates AUC due to shared images between adjacent nodes

**Threshold optimization**: For each fold combination, find τ that maximizes F1 on precision-recall curve. Reported in ± range due to instability (std >0.05 across seeds).

**Feature importance** (via Random Forest on fused set):
- Barranco: betweenness > closeness > degree > clustering > avg_neighbor_degree
- La Victoria: closeness > betweenness > degree > avg_neighbor_degree > clustering

**Output**: `classifier_comparison.csv` (18 rows: 9 combos × 2 schemes) + `predictions.pkl` (out-of-fold probabilities for Paso 7).

### Step 6: Diagnostic Plots (`Paso7_diagnostic_plots.py`)

For the best **fused** combination (enforced by project rule):

1. **Metric vs. threshold**: Precision/Recall/F1 curves, optimal threshold marked
2. **ROC + PR comparison**: All three feature sets under one classifier, spatial CV only
3. **Confusion matrix**: TP/TN/FP/FN at optimal threshold
4. **Graph real vs. predicted**: Two panels, same street network, real labels and model predictions colored
5. **Error map**: Single panel, errors decomposed—TP (dark red), TN (gray), FP (orange), FN (bright red, larger size)
6. **Feature importance**: Random Forest importance, visual vs. structural totals + breakdown of structural features

**Geographic encoding**: All maps use the street network as background, nodes colored by class. Helps validate whether errors cluster spatially or are random.

### Step 7: Robustness Validation (`Paso8_robustness_checks.py`)

**Test 1: Spatial leakage**: 
- GroupKFold CV (spatial blocks) vs. StratifiedKFold (random)
- Gap >0.05 signals significant leakage

**Test 2: Threshold stability**: 
- 10 random seeds, each reporting optimal τ
- If std(τ) > 0.05, report as range not point estimate

**Test 3: Stacking ensemble**: 
- Separate base models per modality (visual and structural)
- Meta-learner combines their probabilities
- Compare to simple concatenation

**Result**: Stacking provides marginal gains (~0.03 AUC) in Barranco, none in La Victoria → dismiss for simplicity.

### Step 8: Qualitative Validation (`Paso9_qualitative_samples.py`)

Generates 3 grids of real Street View images:
1. Barranco only (N_peligroso + N_seguro)
2. La Victoria only
3. Combined (4 rows: Barranco-high-crime, Barranco-safe, La Victoria-high-crime, La Victoria-safe)

Images bordered red (high-crime) or green (safe) per true label. Human inspection confirms whether "peligroso" neighborhoods visually differ from "seguro" ones within and across districts.

## Key Findings

### Main Results (Spatial Cross-Validation, Primary Metric)

| Metric | Barranco | La Victoria |
|--------|----------|-------------|
| **Best Combo (Fused)** | fused + random_forest | fused + random_forest |
| **AUC** | 0.667 | 0.717 |
| **F1 (τ=0.5)** | 0.502 | 0.092 |
| **F1 (τ_optimal)** | 0.640 | 0.373 |
| **Optimal Threshold** | 0.40 | 0.23 |
| **Precision** | 0.69 | 0.53 |
| **Recall** | 0.84 | 0.73 |

**Note**: Barranco AUC (0.667 fused) is lower than structural-only (0.792), but we enforce **fused as project rule** for consistency with multimodal research question, accepting ΔAUC=−0.125.

### Visual vs. Structural Feature Contributions

**Barranco** (compact, homogeneous):
- Structural features alone: AUC 0.792, F1 0.693 (BEST GLOBAL)
- Reason: Street network centrality (betweenness, closeness) is a strong proxy for commercial activity and foot traffic
- Visual signals: Provide marginal improvement or noise

**La Victoria** (heterogeneous, mixed commercial):
- Visual+structural (fused): AUC 0.717, F1 0.373 (BEST)
- Visual-only: AUC 0.708, F1 0.351 (close second)
- Structural-only: AUC 0.533, F1 0.092 (poor)
- Reason: Gamarra commercial district has high visual crowding but moderate degree; topology alone misses this

**Interpretation**: Utility of visual modality depends on urban morphology. In simple grids, structure suffices; in mixed-use zones, appearance matters.

### Feature Importance Hierarchy

Both districts agree: **Closeness/betweenness >> degree >> clustering**

This confirms crime pattern theory (Brantingham & Brantingham): risk concentrates at **high-flow locations** (accessible, well-traveled), not just high-degree nodes.

### Cross-District Robustness

The same two features (closeness, betweenness) dominate in both districts, suggesting the pattern generalizes across Lima neighborhoods of different sizes and socioeconomic profiles.

## Robustness Checks and Limitations

### Spatial Leakage (Paso 8)

Neighborhoods are ego-graphs that spatially overlap: two adjacent nodes share the same crime points (and images) in their 20-image tensors.

**Gap (Random K-fold − Spatial GroupKFold)**:
- Barranco: ΔAUC = +0.197 (0.834 → 0.636 spatial)
- La Victoria: ΔAUC = +0.195 (0.900 → 0.705 spatial)

**Implication**: Reported AUCs (0.667 fused Barranco, 0.717 La Victoria) are the **conservative, leakage-free estimates** suitable for publication. Earlier runs with random CV were inflated.

### Threshold Instability

**Barranco**: τ = 0.40 ± 0.063 (across 10 seeds)
**La Victoria**: τ = 0.23 ± 0.089

Variation exceeds 0.05, so thresholds reported as **ranges not point values** in thesis.

### Data Coverage

| District | Crime Points | With Images | Coverage |
|----------|--------------|-------------|----------|
| Barranco | 1,106 | 985 | 89% |
| La Victoria | 3,807 | 864 | 22.7% |

La Victoria's low coverage (missing northern residential zone, outskirts) risks sampling bias toward central Gamarra. Mitigation: depth-adaptive ego-graphs find neighbors up to 150m away, covering sparse areas.

### Imbalanced Classes

La Victoria: 11.2% high-crime. At τ=0.5, model rarely predicts "crime" (f1_default=0.092 near 0). Remedy: optimal threshold drops to 0.23, recovering recall at cost of false alarm rate (precision 0.53).

### Stacking Ensemble

Tested as alternative to simple concatenation. Results:
- Barranco: marginal gain (+0.028 AUC), adds complexity
- La Victoria: no gain (−0.009 AUC)
**Decision**: Dismissed; simple fusion sufficient.

### Methodological Trade-offs

**Fused vs. Best Global**:
- Barranco: "fused" ranks #3, sacrificing ΔAUC=−0.125, ΔF1=−0.053 vs. structural-only
- La Victoria: "fused" ranks #1, no trade-off

Justified as deliberate project design: confirm multimodal fusion helps **despite** not always winning individual-district benchmarks.

## Critical Design Decisions

### 1. ID Normalization: String Format "X.0"

**Issue**: Crime IDs in Street View folders came from `float(id)` cast: `"19833096.0"`. CSVs may have IDs as int `19833096` or string `"19833096"`.

**Solution**: Canonicalize all IDs to `"19833096.0"` using:
```python
def normalize_crime_id(raw):
    val = float(raw)
    if val == int(val):
        return f"{int(val)}.0"
    return str(val)
```

**Why**: Silent join failure risk. Without this, join would detect 0% overlap if CSV IDs were ints, silently losing all data.

### 2. Filter Candidates by Image Availability BEFORE Sampling

**Bug fixed**: Original code sampled 5 random crime points from ego-graph, then tried to find images. If point had no image, filled with zeros (768 zeros). With 22.7% coverage in La Victoria, average neighborhood got 3.87/5 zero-vectors, diluting mean toward zero and corrupting cosine-similarity Laplacian.

**Fix**: Filter candidates by `crime_id in crime_id_to_paths.keys()` BEFORE sampling. Prevents zero-vector dilution entirely.

**Impact**: La Victoria AUC jumped from 0.72 to 0.87 post-fix.

### 3. GMM Dual Labeling with Zero-Exclusion

**Problem**: Many nodes/neighborhoods have zero crime. Standard 2-component GMM might collapse the "safe" component to a spike at zero, losing discrimination.

**Solution**: Fit GMM on `values[values > 0]` (excluding zeros), then classify all points (including zeros) using the fitted model.

**Trade-off**: Sacrifices information about "how safe" a location is (since zeros aren't modeled), but prevents component collapse.

### 4. Global PCA Fit on All 30k+ Images, Not Per-Neighborhood

**Reasoning**: With N_neighborhood=400 and d=768, fitting PCA on 20 samples per neighborhood gives unstable covariance estimates. Instead, fit once on all 30,866 embeddings, then use transform() on per-neighborhood data.

**Decision**: This is a form of transfer learning—we trust global structure more than local neighborhood structure at small sample sizes.

### 5. Laplacian Eigenvectors: Skip First (Trivial) Mode

**Theory**: First eigenvector of Laplacian is a constant (all 1s), carries no information. Standard spectral methods skip it.

**Problem in this context**: First eigenvector, when projected, would be the "average appearance" of the 20 images. This is actually predictive! But spectral theory says skip it.

**Trade-off**: We respect theory and skip it, accepting AUC≈0.5. This is why mean-pooling baselines dominate.

### 6. Enforce "Fused" Feature Set Despite Lower Performance in Barranco

**Barranco result**: Structural-only (AUC 0.792) beats fused (AUC 0.667).

**Project rule**: Always report and use the best **fused** combination, not the global best. Fused ranks #3 in Barranco.

**Justification**: The research question is "can visual+structural together predict crime?" not "which modality wins?" Reporting "structure alone wins" would be factually correct but deflates the multimodal narrative. Explicitly document the cost (ΔAUC=−0.125) so reviewers can see the trade-off.

### 7. GroupKFold Spatial Blocks Over StratifiedKFold

**Discovery**: Random K-fold inflates AUC by ~0.2 (leakage from adjacent neighborhoods).

**Fix**: Use GroupKFold with 5 geographic blocks (KMeans on lat/lon). All nodes in a block stay together across folds.

**Cost**: Lower reported AUC, but honest and reproducible. This becomes the **primary metric** in thesis.

## Reproducibility Notes

### Known Sources of Non-Determinism

1. **Embedding extraction**: GPU float precision varies slightly. Affect: <0.1% of values
2. **KMeans spatial blocks**: Random initialization. Mitigation: fixed seed (SEED=42)
3. **Cross-validation splits**: StratifiedKFold shuffle with seed=SEED, but GroupKFold is deterministic given groups
4. **Threshold optimization**: Numerical precision in precision-recall curve. Affect: τ stable to 0.01

### Exact Reproduction

To reproduce exact numbers:

```bash
export PYTHONHASHSEED=42
export CUDA_VISIBLE_DEVICES=0  # Single GPU only

python Paso6_final_classifier.py --distrito all --year 2016
python Paso7_diagnostic_plots.py --distrito all --year 2016
python Paso8_robustness_checks.py --distrito all --year 2016
```

Expect output metrics to match documented results within ±0.005 (AUC), ±0.01 (F1).

## Future Work

### Immediate Extensions (Minimal Effort)

1. **Temporal validation (2019, 2021)**: Re-scrape images for later years, measure cross-year consistency
2. **Nonlinear classifiers**: Test XGBoost, neural networks with learned fusion layer
3. **Explainability**: SHAP values for feature importance, saliency maps for what parts of images matter
4. **Neighborhood-level policies**: Simulate policy interventions (e.g., increase foot traffic) and predict crime impact

### Medium-term Research

1. **Why did visual matter less in Barranco?** Qualitative analysis of visual feature importance using layer-wise relevance propagation (LRP)
2. **Cross-city transfer**: Train in Lima, test in other Peruvian or Latin American cities
3. **Active learning**: Which neighborhoods' images would most improve the model if labeled?

### Structural Limitations to Address

1. **Imbalanced learning**: Explore cost-sensitive training, focal loss, synthetic oversampling for La Victoria's 11% minority class
2. **Coverage bias**: Determine if La Victoria's missing 77% of points are systematically safer/more dangerous
3. **Image quality**: Filter for time-of-day artifacts (night, rain) that may bias visual features
4. **Street-level vs. block-level**: Cluster nodes into larger "blocks" to reduce spatial correlation?
